# Goal 在项目根目录建立`doc` 文件夹,里面存放doc文档. 其中包括以下几部分

1. 项目各个部分的说明: 以文件/文件夹的程度说明

2. 各个层级的说明:以抽象程度说明,譬如: 用户层 内核层 控制层等等

3. 重点实现的最小模型: 以教学为目的

4. Q&A:

    1. 数据如何流动?

    2. 内核态在做什么?

    3. 用户态在做什么?

    4. 画出数据流(用文本图)

    5. 这个工具在监控什么?

    6. eBPF程序如何挂载, 挂载在哪?

    7. 如何实现数据持久化?

> 每一个部分可以建立文件夹或单独的一个文件说明, 不限制文件类型:
> 比如有些问题可以跑一个最小实现, 搭配代码,注释,脚本等等一些东西去说明.
