# eBPF-Sentinel 项目提示词模板

## 项目概述

```yaml
项目名称: eBPF-Sentinel
项目类型: 基于 eBPF 的实时系统监控与治理平台
技术栈:
  - eBPF (Extended Berkeley Packet Filter)
  - Go (Golang)
  - C (内核编程)
  - SQLite (数据持久化)
  - WebSocket (实时通信)
  - Gin (Web 框架)
```

## 核心功能

```yaml
监控能力:
  进程监控:
    - 捕获 execve 系统调用
    - 记录 PID、PPID、进程名、命令参数
    - 基于 tracepoint 实现
    
  网络监控:
    - 捕获所有 IPv4 数据包
    - 记录 IP、端口、协议、方向、包大小
    - 基于 TC (Traffic Control) 实现
    - 支持采样机制（每100包采样1个）
    
  系统指标:
    - CPU 使用率
    - 网络入站/出站速度
    - 基于 gopsutil 实现

治理功能:
  - 白名单过滤（进程名、IP、端口）
  - 监控开关动态控制
  - 进程终止（SIGTERM/SIGKILL）
  
数据展示:
  - Web 前端实时展示
  - WebSocket 实时推送
  - 历史数据查询
  - 进程拓扑图
```

## 技术架构

```yaml
架构分层:
  展示层:
    技术: HTML5/JavaScript/WebSocket
    功能: 实时事件展示、进程列表、网络流量图、系统指标图表
    
  应用层:
    技术: Gin/GORM
    组件:
      - HTTP API (RESTful)
      - WebSocket Hub
      - Policy Manager (策略管理)
      - Event Processor (事件处理器)
      
  插件层:
    技术: Go Interface
    组件:
      - Execve Plugin (进程监控)
      - Network Plugin (网络监控)
      - System Monitor Plugin (系统监控)
      - Plugin Manager (插件管理器)
      
  eBPF运行时层:
    技术: cilium/ebpf, bpf2go
    功能:
      - eBPF 对象加载
      - 程序挂载
      - Map 操作
      - Ring Buffer 读取
      
  内核层:
    技术: eBPF 虚拟机
    组件:
      - Verifier (验证器)
      - JIT Compiler
      - BPF Maps (Ring Buffer, Hash, Array, Per-CPU Array)
      - Tracepoint 子系统
      - TC 子系统
      
  硬件/系统层:
    事件源:
      - 进程执行 (execve)
      - 网络数据包 (sk_buff)
      - CPU 状态
      - 网络接口统计
```

## 关键技术点

```yaml
核心技术:
  eBPF程序设计:
    - SEC宏定义挂载点
    - bpf_printk调试
    - bpf_get_current_pid_tgid获取PID
    - bpf_get_current_comm获取进程名
    - bpf_probe_read_user_str读取用户空间数据
    
  BPF Maps:
    - Ring Buffer: 高效事件传输（内核→用户）
    - Hash Map: 白名单存储
    - Array Map: 开关状态
    - Per-CPU Array: 采样计数器
    
  数据传输:
    - bpf_ringbuf_reserve: 预留空间
    - bpf_ringbuf_submit: 提交事件
    - bpf_ringbuf_discard: 丢弃事件
    
  Go-eBPF集成:
    - bpf2go代码生成
    - rlimit.RemoveMemlock解除内存限制
    - link.Tracepoint挂载tracepoint
    - link.AttachTCX挂载TC
    - ringbuf.NewReader读取事件
    - unsafe包高效解析二进制数据
    
  性能优化:
    - 采样机制减少事件量
    - 白名单过滤减少噪音
    - 零拷贝数据传输
    - 异步事件处理
    - goroutine并发处理
```

## 数据流设计

```yaml
事件数据流（上行）:
  流程: 硬件事件 → 内核钩子 → eBPF程序 → BPF Maps → Go程序 → 数据库 → WebSocket → 前端
  
  详细步骤:
    1. 硬件层: 进程执行/网络包到达
    2. 内核层: 触发tracepoint/TC钩子
    3. eBPF层: 程序执行，写入Ring Buffer
    4. 运行时层: Go程序读取Ring Buffer
    5. 应用层: 解析事件，存储数据库，广播WebSocket
    6. 展示层: 前端接收并展示

控制流（下行）:
  流程: 前端 → HTTP API → Policy Manager → eBPF Maps → 内核生效
  
  详细步骤:
    1. 展示层: 用户操作前端界面
    2. 应用层: HTTP API接收请求
    3. 应用层: Policy Manager更新状态
    4. 运行时层: 写入BPF Maps
    5. 内核层: eBPF程序读取Maps，调整行为
```

## 项目亮点

```yaml
创新点:
  - 基于eBPF的无侵入监控
  - 内核态+用户态协同设计
  - 插件化架构支持扩展
  - 实时WebSocket推送
  - 动态策略配置
  
技术优势:
  - 高性能: 内核态执行，JIT编译
  - 安全性: Verifier验证，不会崩溃内核
  - 灵活性: 动态加载/卸载，运行时更新
  - 低开销: 采样机制，白名单过滤
  - 易用性: Web界面，RESTful API
  
应用场景:
  - 安全监控: 异常进程检测、网络行为分析
  - 运维监控: 服务启动监控、流量分析
  - 故障排查: 进程追踪、网络连接分析
```
